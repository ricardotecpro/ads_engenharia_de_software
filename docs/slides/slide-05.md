# Slide 05 - Modelagem e UML

## Slide 1: Título
- **Título**: Modelagem de Software
- **Subtítulo**: Aula 05 - Diagramas e UML

## Slide 2: Por que modelar?
- Construir sem planta = Desastre.
- Modelos ajudam a:
    - Entender o problema.
    - Comunicar a solução.
    - Documentar decisões.

## Slide 3: UML (Unified Modeling Language)
- Padrão internacional.
- Linguagem visual (caixas, linhas, setas).
- Independente de linguagem de programação.

## Slide 4: Diagrama de Caso de Uso
- Foco nas funcionalidades externas.
- **Componentes**:
    - Ator (Usuário).
    - Caso de Uso (Funcionalidade).
    - Relacionamento (Linha).

## Slide 5: Diagrama de Classes
- Foco na estrutura interna.
- Mostra os objetos do sistema.
- **Estrutura**:
    - Nome da Classe.
    - Atributos (Dados).
    - Métodos (Ações).

## Slide 6: Resumo
- Use Caso de Uso para falar com o cliente.
- Use Classes para falar com os programadores.
