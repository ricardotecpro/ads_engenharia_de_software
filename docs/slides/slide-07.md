# Slide 07 - Controle de Versão (Git e GitHub)

## Slide 1: Título
- **Título**: Controle de Versão
- **Subtítulo**: Aula 07 - Git e GitHub

## Slide 2: Por que Versionar?
- Evitar `arquivo_final_v2.doc`.
- "Máquina do tempo" do código.
- Trabalho em equipe sem conflitos.
- Backup na nuvem.

## Slide 3: Conceitos
- **Repositório**: A pasta do projeto gerenciada pelo Git.
- **Commit**: Uma "foto" do estado atual (Save point).
- **Branch**: Uma linha do tempo paralela.
- **Merge**: Juntar duas linhas do tempo.

## Slide 4: Fluxo de Trabalho
1. **Edit** (Modificar arquivos).
2. **Stage** (`git add` - Escolher o que salvar).
3. **Commit** (`git commit` - Salvar localmente).
4. **Push** (`git push` - Enviar para a nuvem/GitHub).

## Slide 5: Git vs. GitHub
- **Git**: Motor (Local).
- **GitHub**: Garagem (Nuvem).

## Slide 6: Resumo
- Nunca trabalhe sem Git.
- Commit frequentemente.
- Mensagens claras.
